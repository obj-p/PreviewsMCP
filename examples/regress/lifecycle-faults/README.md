# Session Lifecycle and Failure Isolation

This package supports four separate experiments on one iOS simulator. Each
fault family is a separate SwiftPM target so the missing-symbol object cannot
contaminate the replacement, slow-render, or crash-agent controls.

## Same-device replacement

Start `SessionAPreview.swift` with `--detach`, then immediately start
`SessionBPreview.swift` on the same simulator. Repeat through both `run` and
`variants`. The result must be deterministic replacement or deterministic
fail-fast; it must not wait for an orphaned agent connection.

## Missing JIT symbol

`MissingSymbolPreview.swift` declares and calls a symbol that does not exist.
The package can compile the reference into an object, but JIT materialization
must return a session-scoped unresolved-symbol error. Afterwards, `list` and a
normal preview start must still work through the same daemon.

## Slow render

`SlowRenderPreview.swift` blocks view initialization for eight seconds. This is
long enough to exercise setup/render heartbeat messages without turning the
fixture into a multi-minute test.

## Agent crash

`AgentCrashPreview.swift` renders normally and traps only when its button is
pressed. A physical tap deliberately kills the preview agent. The daemon should
report that failure, clean up the session, and remain responsive.

The bad-slice and invalid-dylib inputs generated by `../binary-frameworks` cover
startup failures before the view is rendered.
